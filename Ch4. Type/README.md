## 자료형의 종류와 특징
자료형이란 어떤 데이터를 저장할 지 표현하는 것으로, Go언어의 자료형으로는 bool, 문자열, 정수형, 실수, 복소수, 기타 타입이 있다.
``` := ``` 를 사용하면 자료형 추론이 가능하며, 실수 값은 float32로 자동 할당된다.
Go에서는 null을 nil로 표현한다. 

## import "unsafe"
Go 언어에서 자료형의 크기를 알 수 있는 방법은 바로 ``` unsafe.Sizeof(변수) ``` 를 사용하는 것이다.

## 정수 타입
uintptr은 포인터의 비트 패턴을 할당할만한 크기의 자료형이다. 주로 포인터의 주소를 할당할 때 사용한다.

|자료형|선언|크기(byte)|
|---|---|---|
||int|n비트 시스템에서 n비트|
||int8|1|
|**정수형 (음수포함)**|int16|2|
||int32|4|
||int64|8|
||uint|n비트 시스템에서 n비트|
||uint8|1|
|**정수형 (0, 양수)**|uint16|2|
||uint32|4|
||uint64|8|
||uintptr|8|

## 실수 및 복소수 타입
precision에 따라 자료형을 선택해야 한다.
복소수 선언은 3+4i 처럼 선언할 수 있다.

|자료형|선언|크기(byte)|
|---|---|---|
|실수|float32|4|
||float64|8|
|복소수|complex64|8|
||complex128|16|

## 문자열 타입
string으로 선언한 문자열 타입은 immutable 타입으로, 값을 수정할 수 없다.
Ex) ```var str string = "hello"```와 같이 선언하고 ```str[2] = 'a'```로 수정이 불가하다.
|자료형|선언|크기(byte)|
|---|---|---|
|문자열|string|16|

## 기타 타입
byte는 unit8과 똑같은 자료형이다. 바이트 값을 8비트 부호 없는 정수 값과 구별하는 데 사용된다.
rune은 int32와 똑같은 자료형이다. 관례상 문자 값을 정수 값과 구별하기 위해 사용한다.
|자료형|선언|크기(byte)|
|---|---|---|
|정수(0, 양수)|byte|1|
|정수|rune|4|

## 문자열의 표현
Go언어에서는 문자열을 표현하는 데 두 가지 방법이 있다.
+ Back Quote (`문자열`)을 이용하는 방법, Back Quote로 둘러 싸인 문자열은 Raw String Literal이라고 부르며 어느 기호든 문자열 자체로 인식된다.
+ 이중인용부호("문자열")을 이용하는 방법, 이중인용부호로 둘러싸인 문자열은 Interpreterd String literal이라고 부른다. 예를 들어 \n과 같은 기호가 있으면 문자열 자체로 인식되지 않고 개행의 의미로 해석된다.

'+' 연산자는 숫자 뿐만이 아니라 문자열도 합할 수 있다.

## 자료형의 변환
Go 언어에서는 형 변환을 할 때 명시적으로 지정해야 한다. (자동 형 변환 불가)

그렇기 때문에 연산식의 결괏값과 그것을 저장하려는 변수의 자료형이 다를 때 에러가 발생한다.

C언어와는 달리 ```uint(변수이름)```와 같이 강제 형 변환한다.
